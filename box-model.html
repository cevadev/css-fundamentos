Modelo de Caja en CSS

El Modelo de Caja

El modelo de caja es un concepto fundamental en CSS y se refiere a la forma en que CSS presenta elementos en una página web. Básicamente, cada elemento HTML se trata como un cuadro rectangular y cada cuadro tiene cuatro componentes:

Content: Este es el contenido real del elemento, como texto o una imagen. Tomando en cuenta sus dimensiones width & height
Padding: Este es el espacio entre el contenido y el borde del elemento. El relleno es transparente y separa el contenido del borde, lo que le permite agregar espacio alrededor del contenido.
Border: Esta es la línea que rodea el relleno y el contenido del elemento. El borde puede tener un ancho, estilo y color que se puede configurar con CSS.
Margin: Este es el espacio fuera del borde. Separa el elemento de otros elementos en la página y se puede usar para agregar espacio entre elementos.

Cada uno de estos componentes del modelo de caja puede tener su propio tamaño y estilo, y se pueden manipular con CSS para controlar el diseño de los elementos en una página web. El tamaño del cuadro está determinado por la altura y el ancho del contenido, más el relleno, el borde y el margen agregados al elemento.

Dentro del modelo de caja hay que considerar que una caja tiene 4 lados:

1. Arriba (top)
2. Derecha (right)
3. Abajo (bottom)
4. Izquierda (left)

Cajas de Linea vs Cajas de Bloque

En CSS, "cajas de linea" y "cajas de bloque" se refieren a las diferentes formas en que un elemento HTML puede mostrar sus bordes.

Una caja de linea ocupa el espacio necesario para mostrar su contenido
Las cajas de linea no tienen dimensiones modificables (alto, ancho)
Las cajas de lineas permiten otros elementos a su lado
En las cajas de linea padding y margin solo empujan a los elementos adyacentes en horizontal, NUNCA EN VERTICAL

Las cajas de bloque ocupan todo el ancho disponible, lo que genera saltos de linea
Las cajas de bloque tienen dimensiones modificables (alto, ancho)
Las cajas de bloque No permiten elementos a su lado (aunque especifiquen un ancho, siguen ocupando todo el espacio disponible)

Propiedad Display

La propiedad "display" en CSS se usa para especificar el tipo de diseño que debe tener un elemento HTML en particular. Determina cómo se debe mostrar el elemento en la página web, como en línea, bloque, bloque en línea, tabla, flex, cuadrícula, etc. El valor de la propiedad "display" puede afectar en gran medida el diseño y el comportamiento del elemento y los elementos que lo rodean. Algunos valores comunes incluyen:

none el elemento es removido del documento html, es decir, se hace invisible
block el elemento ocupará todo el ancho de su contenedor principal y creará un nuevo contexto de formato de bloque.
inline El elemento solo ocupará el ancho necesario y no creará un nuevo contexto de formato de bloque.
inline-block el elemento actuará como un elemento en línea, pero puede tener un ancho y una altura específicos.
flex El elemento se convertirá en un contenedor flexible, lo que permite que sus elementos secundarios se organicen de manera flexible.
grid El elemento se convertirá en un contenedor de cuadrícula, lo que permite que sus elementos secundarios se organicen en forma de cuadrícula.

Propiedad Visibility

La propiedad de "visibilidad" en CSS se usa para especificar si un elemento debe ser visible o no. Los valores de la propiedad "visibilidad" determinan si el elemento ocupará espacio en el diseño de una página web y si será visible para el usuario. Los posibles valores de la propiedad "visibilidad" son:

viisble El elemento es visible y ocupa espacio en el diseño de la página web. Este es el valor predeterminado.
hidden El elemento no es visible pero aún ocupa espacio en el diseño de la página web.
collapse este valor solo se aplica a los elementos de la tabla (p. ej. tr, td, th). Cuando se aplica un elemento con visibilidad de "contraer" a una celda de tabla, la celda se elimina de la tabla y no ocupa espacio en el diseño.

Propiedad Overflow

La overflowpropiedad CSS se usa para controlar lo que sucede cuando el contenido desborda su contenedor. Especifica si recortar el contenido, agregar barras de desplazamiento o mostrar el contenido fuera del contenedor. Sus propiedades son:

visible: el contenido no se recorta y puede mostrarse fuera del contenedor.
hidden: El contenido se recorta y se oculta cualquier desbordamiento.
scroll: El contenido se recorta, pero se agrega una barra de desplazamiento para ver el resto del contenido.
auto: El contenido se recorta, pero se agrega una barra de desplazamiento solo cuando es necesario.

Al usar la overflowpropiedad, es importante asegurarse de que el contenedor tenga una altura o un ancho definidos, de lo contrario, es posible que el comportamiento de desbordamiento no funcione según lo previsto.

Box Size

La propiedad CSS box-sizing se usa para controlar cómo se calcula el ancho y alto total de un elemento.

De forma predeterminada, la box-sizing propiedad se establece en content-box, lo que significa que el ancho y el alto especificados de un elemento solo incluyen el contenido, pero no el relleno, el borde o el margen.

Sin embargo, cuando la box-sizing propiedad se establece en border-box, el ancho y el alto especificados de un elemento incluyen el contenido, el relleno y el borde, pero no el margen. Esto puede ser útil para mantener un tamaño de elemento uniforme incluso al agregar relleno y bordes.

La box-sizing propiedad también se puede establecer en inherit, lo que hace que el elemento herede el box-sizing valor de su padre.

Propiedad Float & Clear

La propiedad flotante de CSS le permite colocar un elemento a la izquierda o a la derecha de su contenedor principal, lo que permite que otro contenido fluya a su alrededor. Se usa comúnmente para crear diseños con múltiples columnas o para colocar imágenes y otros medios en relación con el texto.

Cuando aplica la propiedad flot a un elemento, también debe aplicar una propiedad de ancho para definir el tamaño del elemento. De forma predeterminada, los elementos flotantes se eliminan del flujo de documentos normal y no ocupan espacio en el contenedor. Esto a veces puede causar problemas con el diseño, como cuando la altura del contenedor no se ajusta para acomodar los elementos flotantes.

La propiedad clear de CSS se usa para controlar el comportamiento de diseño de los elementos que siguen a un elemento flotante. Cuando se borra un elemento, se mueve hacia abajo debajo de los elementos flotantes que lo preceden en el flujo del documento. La propiedad clear tiene cuatro valores posibles: izquierda, derecha, ambas y ninguna.

El valor de la izquierda borra los elementos flotantes a la izquierda del elemento, el valor de la derecha borra los elementos flotantes a la derecha del elemento y el valor de ambos borra los elementos flotantes a ambos lados del elemento. El valor none significa que el elemento no borra ningún elemento flotante y se colocará lo más cerca posible del contenido anterior. El uso de la propiedad clear en combinación con la propiedad float le permite crear diseños más complejos y controlar la posición de los elementos en la página.

Colapso de Margenes verticales

El colapso de márgenes es un comportamiento en CSS que ocurre cuando dos márgenes verticales adyacentes (márgenes superior e inferior) de elementos se tocan, lo que resulta en la creación de un solo margen que es igual al máximo de los dos márgenes.

En otras palabras, cuando dos elementos se apilan verticalmente, el margen entre ellos se contrae en un solo margen igual al mayor de los dos márgenes. Este es el comportamiento predeterminado en CSS y es importante comprenderlo para evitar problemas de diseño inesperados.

Hay formas de evitar el colapso de los márgenes, y es utilizando simpre margin-top o margin-bottom a nuestros elmentos . Al hacerlo, puede asegurarse de que los márgenes entre los elementos no se colapsen y que su diseño permanezca como pretendía.
